# 隐私政策 — VaultSearchExtension (CmdPal)

生效日期：2026-06-17

**概述**

- 本扩展为 Command Palette 的本地文件与 Obsidian Vault 搜索插件。
- 扩展无任何网络访问行为，所有主要的数据处理在用户设备本地执行，用以提供索引与搜索功能。

**我们收集的信息（本地处理）**

- 文件元数据：文件路径、文件名、扩展名、大小、最后修改时间等，用于索引与排序。
- 文件内容预览：为构建搜索索引会在内存中保留每个文件的内容摘要。
- 标题与标签：从文件中提取用于索引的标题和标签。
- 用户设置：扩展的配置项配置文件与 Command Palette 配置行为一致。

**特殊读取行为**

- 若启用 Obsidian Vault 功能，扩展会读取本地 Obsidian 配置：`%APPDATA%\Obsidian\obsidian.json` 以实现自动发现仓库功能。

**数据的存储与传输**

- 索引数据（倒排索引、文件索引等）仅保存在进程内存中，不会持久化到网络或远程服务器；索引随进程生命周期存在或在重建索引时重新生成。
- 用户设置保存在本地 JSON 文件。
- 仓库代码中未包含任何远程遥测、崩溃上报或网络上报逻辑，因此扩展默认不会向外部发送用户文件或索引数据。

**第三方与库**

- 扩展可能使用本地第三方库（例如 `EzPinyin`）做文本处理，第三方库仅在本地运行，不改变隐私行为。

**用户控制**

- 若不希望某些目录被索引，请勿在设置中添加该目录；可通过设置删除 `Vault Path` 或关闭 Obsidian 功能停止相关索引。
- 要清除本地设置，请删除卸载扩展；内存索引会在扩展进程结束时释放。

**安全**

- 扩展读取本地文件，请按照常规 Windows 安全实践管理文件与账号权限。
- 日志通过宿主 Command Palette 接口记录，在共享日志前请检查并移除敏感信息（例如完整路径）。

**政策变更**

- 若隐私实践发生实质性变更（例如引入远程上报），此文档将更新并注明生效日期。

**联系方式**

- 如有隐私相关问题或建议，请通过项目仓库 Issue 提交：https://github.com/chanbeiyu/CmdPal/issues
